# سياسة تشغيل MukallaStay (SOP Booking Flow)

1. عندما يأتي حجز جديد (PENDING):
   - يتلقى الفريق إشعارًا.
   - يتواصل مع الفندق لتأكيد أو رفض الحجز.

2. عند تأكيد الفندق:
   - الموظف يضغط "Confirm" من لوحة الإدارة.
   - النظام يرسل للعميل رسالة تأكيد مع رقم الحجز.

3. إذا الفندق لم يرد خلال 24 ساعة:
   - الطلب يُلغى تلقائيًا أو يُعرض على فندق آخر.

4. عند اكتشاف تكرار حجز من نفس العميل عدة مرات:
   - الحجز يُعلَّم كاحتيالي (fraud_flag=1) ويُراجع يدويًا.

5. كل عملية (تأكيد أو إلغاء) تُسجَّل في جدول audit_logs.
